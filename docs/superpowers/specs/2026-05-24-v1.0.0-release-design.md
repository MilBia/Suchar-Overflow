# Release 1.0.0 — Design Spec

**Date:** 2026-05-24
**Approach:** Sequential gate — each gate must pass before the next starts.

---

## Overview

Suchar Overflow is currently at `1.0.0-beta.1`. This release promotes it to `1.0.0`.
No new features are in scope. The goal is verification, cleanup, and a clean tag.

---

## Gate 1 — Tests

Run both test suites inside the local Docker containers (never directly in `.venv`):

```bash
just test        # unit/integration — must be 100% green
just test-e2e    # Playwright E2E — must be 100% green
```

**Pass criteria:** Zero failures in both suites.
**On failure:** Fix the failing test(s) before advancing.

---

## Gate 2 — Translations

Scan `locale/` for empty `msgstr ""` entries in all `.po` files (both `pl` and `en`).

- If gaps exist: fill them **manually**.
- After filling: run `compilemessages` inside Docker to rebuild `.mo` files.
- Commit updated `.po` and `.mo` files.

**Pass criteria:** No empty `msgstr ""` in any `.po` file.

---

## Gate 3 — Production stack spin-up

Build and start `docker-compose.production.yml` with minimal dummy env values:

- `.envs/.production/.django` — fake secret key, fake domain, fake admin URL
- `.envs/.production/.postgres` — fake credentials

**Verify:**
- All containers reach running/healthy state (Django, Postgres, Redis, Traefik, Nginx, rqworker)
- Django startup logs show no errors (missing settings, import failures, migration issues)

Tear down after verification. No real domain or SSL cert required.

**Pass criteria:** Stack starts cleanly, no errors in Django startup logs.

---

## Gate 4 + 5 — Version bump + CHANGELOG (one commit)

### Version bump
- `pyproject.toml`: `version = "1.0.0-beta.1"` → `"1.0.0"`
- `README.md`: badge `version-1.0.0--beta.1-blue` → `version-1.0.0-blue`

### CHANGELOG
- Create `CHANGELOG.md` at project root
- `[1.0.0]` entry built from full `git log`, grouped into:
  - **Funkcje** (Features)
  - **Poprawki** (Fixes)
  - **Refaktoryzacja** (Refactoring)
  - **Infrastruktura / DevOps**
  - **Dokumentacja**
- Written in Polish (project's primary language)

Both changes in a single commit: `chore: bump version to 1.0.0 and add CHANGELOG`.

---

## Gate 6 — Git tag

Create an annotated tag on the version bump commit and push it:

```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

No GitHub release — this is a personal project.

**Pass criteria:** Tag `v1.0.0` visible on `origin`.

---

## Out of scope

- New features
- GitHub Releases / release notes page
- Deployment to a live server
