# v1.0.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Promote Suchar Overflow from `1.0.0-beta.1` to a clean, tagged `1.0.0` release by passing six sequential verification gates.

**Architecture:** Sequential gate model — each gate must pass fully before the next begins. No gate is skipped even if it looks fine at a glance. Tests run in local Docker containers, never in the host `.venv`.

**Tech Stack:** Django 5.2, Docker Compose, just, pytest, pytest-playwright, gettext, git

## Gate 2: Translations

**Files:**
- Possibly modify: `locale/pl/LC_MESSAGES/django.po`
- Possibly modify: `suchar_overflow/templates/registration/password_reset_email.html`

**Context:** The app supports two languages: Polish (`pl`) and English (`en`). `en` strings with empty `msgstr` are functionally correct because Django falls back to the `msgid`, which is already in English. Only `pl` gaps are blocking.

- [ ] **Step 1: Find untranslated pl strings**

```bash
python3 -c "
import re
path = 'locale/pl/LC_MESSAGES/django.po'
with open(path) as f:
    content = f.read()
pattern = r'msgid \"(.+?)\"\nmsgstr \"\"\s*\n(?!\"|msgstr)'
matches = re.findall(pattern, content)
print(f'Untranslated pl strings: {len(matches)}')
for m in matches:
    print(f'  - {repr(m)}')
"
```

Expected output (known state at time of writing):

```
Untranslated pl strings: 1
  - 'The'
```

- [ ] **Step 2: Fix the "The" string in the password reset email template**

The template at `suchar_overflow/templates/registration/password_reset_email.html` line 13 fragments a sentence into three pieces: `{% translate "The" %} Suchar Overflow {% translate "team" %}`. Polish has no definite article, so translating "The" in isolation is incorrect. Fix by collapsing the line into a single translatable phrase.

Edit `suchar_overflow/templates/registration/password_reset_email.html` line 13:

Before:
```
  {% translate "The" %} Suchar Overflow {% translate "team" %}
```

After:
```
  {% translate "The Suchar Overflow team" %}
```

- [ ] **Step 3: Update the .po file**

Run `makemessages` inside Docker to regenerate the .po files (this removes the orphaned "The" entry and adds the new phrase):

```bash
docker compose -f docker-compose.local.yml run --rm django \
  python manage.py makemessages -l pl -l en
```

- [ ] **Step 4: Add the pl translation for the new phrase**

Open `locale/pl/LC_MESSAGES/django.po` and find the new entry:

```
msgid "The Suchar Overflow team"
msgstr ""
```

Fill in the translation manually:

```
msgid "The Suchar Overflow team"
msgstr "Zespół Suchar Overflow"
```

- [ ] **Step 5: Verify no remaining pl gaps**

```bash
python3 -c "
import re
path = 'locale/pl/LC_MESSAGES/django.po'
with open(path) as f:
    content = f.read()
pattern = r'msgid \"(.+?)\"\nmsgstr \"\"\s*\n(?!\"|msgstr)'
matches = re.findall(pattern, content)
print(f'Remaining untranslated pl strings: {len(matches)}')
for m in matches:
    print(f'  - {repr(m)}')
"
```

Expected: `Remaining untranslated pl strings: 0`

- [ ] **Step 6: Compile messages inside Docker**

```bash
docker compose -f docker-compose.local.yml run --rm django \
  python manage.py compilemessages
```

Expected: no errors, `.mo` files regenerated (but do **not** stage `.mo` files for commit).

- [ ] **Step 7: Run pre-commit to auto-format the template**

```bash
pre-commit run --all-files
```

Run a second time if pre-commit auto-fixed anything:

```bash
pre-commit run --all-files
```

Expected: all hooks pass on the second run.

- [ ] **Step 8: Commit .po and template changes only (no .mo files)**

```bash

        locale/en/LC_MESSAGES/django.po \
        suchar_overflow/templates/registration/password_reset_email.html
```

## Gate 3: Production stack spin-up

**Files:**
- Create temporarily (not committed): `.envs/.production/.django`
- Create temporarily (not committed): `.envs/.production/.postgres`

**Context:** Spin up the production Docker Compose stack with fake credentials to verify all services start and Django boots without errors. Tear down and delete the temp env files afterwards.

- [ ] **Step 1: Create dummy production env files**

Create `.envs/.production/.django`:

```
DJANGO_SETTINGS_MODULE=config.settings.production
DJANGO_SECRET_KEY=fake-secret-key-aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa
DJANGO_ADMIN_URL=admin-test/
DJANGO_ALLOWED_HOSTS=localhost
DJANGO_SECURE_SSL_REDIRECT=False
REDIS_URL=redis://redis:6379/0
FEEDBACK_URL=https://github.com/MilBia/Suchar-Overflow/issues/new
```

Create `.envs/.production/.postgres`:

```
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DB=suchar_overflow
POSTGRES_USER=test_user
POSTGRES_PASSWORD=test_password_123
```

- [ ] **Step 2: Build the production images**

```bash
docker compose -f docker-compose.production.yml build
```

Expected: build completes without errors. This may take several minutes.

- [ ] **Step 3: Start the stack**

```bash
docker compose -f docker-compose.production.yml up -d
```

Expected: all containers start. Check with:

```bash
docker compose -f docker-compose.production.yml ps
```

All services should show `running` or `healthy`.

- [ ] **Step 4: Check Django startup logs for errors**

```bash
docker compose -f docker-compose.production.yml logs django 2>&1 | head -60
```

Expected: no `ERROR`, `CRITICAL`, `ImproperlyConfigured`, or traceback lines. You should see `[gunicorn]` startup messages and migration output.

- [ ] **Step 5: Gate decision**

If all containers are running and logs are clean → proceed to tear down.
If there are errors → investigate, fix, and re-run from Step 2.

- [ ] **Step 6: Tear down the stack**

```bash
docker compose -f docker-compose.production.yml down -v
```

- [ ] **Step 7: Delete the temporary env files**

```bash
rm .envs/.production/.django .envs/.production/.postgres
```

Verify they are gone:

```bash
ls .envs/.production/
```

Expected: only `.django.example` and `.postgres.example` remain.

```markdown
- justfile z komendami skróconymi dla lokalnego i produkcyjnego środowiska

### Dokumentacja
```
